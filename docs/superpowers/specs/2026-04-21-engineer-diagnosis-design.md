# 公務員向けITエンジニア適性診断アプリ 実装仕様書

## 概要

公務員を対象に、ITエンジニアへのキャリアチェンジ適性を診断するWebアプリ。診断結果の詳細閲覧をLINE公式アカウント友だち登録につなげ、TechVillageの見込み顧客リストを獲得する。

**ゴール:** LINE友だち登録数の増加 → 個別相談・スクール入学への送客

---

## 質問設計

全7問・各4択（よくある3点 / ときどきある2点 / あまりない1点 / ほとんどない0点）

| # | 質問文 | 主な採点軸 |
|---|---|---|
| 1 | 同じ部署・業務が続くより、新しいことを学び続けたい | S |
| 2 | 仕事の中で「なぜこの手順なんだろう」と疑問を感じることがある | I |
| 3 | 通勤・時間・場所に縛られない働き方に魅力を感じる | F |
| 4 | 頑張りや成果が給与や評価に反映されてほしい | A |
| 5 | 毎年の定期異動に違和感やもったいなさを感じる | S |
| 6 | 業務をもっと効率化・自動化できればと思うことがある | I |
| 7 | 5年後・10年後も自分のスキルが市場で通用するか不安がある | S / A |

**スコア軸の定義**

| 軸 | タイプ |
|---|---|
| S（Specialist） | 積み上げ型エンジニアタイプ |
| I（Improver） | 改善志向型エンジニアタイプ |
| F（Flexible） | 自由志向型エンジニアタイプ |
| A（Achiever） | 成果実感型エンジニアタイプ |

**タイプ判定:** 4軸の合計スコアが最も高いものを診断タイプとして返す。同点の場合は S > I > F > A の優先順位で決定。

## スコアリング仕様

```typescript
// 適性スコア（表示用）: 80〜100 に圧縮
// 目的: どの回答パターンでも高い適性として前向きに受け取れる
const SCORE_MIN = 80
const SCORE_MAX = 100
const RAW_MAX = 21  // 7問 × 3点

function calcDisplayScore(rawTotal: number): number {
  return Math.round(SCORE_MIN + (rawTotal / RAW_MAX) * (SCORE_MAX - SCORE_MIN))
}
// 最低回答(0点) → 80点、満点(21点) → 100点
```

## 診断タイプ定義

### タイプ① 積み上げ型エンジニアタイプ（Specialist）
- **キャッチ:** 専門性を武器にする、コツコツ成長型
- **説明:** 異動のない環境でスキルを積み上げることにやりがいを感じやすいタイプ。知識が資産になる働き方と相性が良い。
- **向いている職種:** Webエンジニア・インフラエンジニア・セキュリティエンジニア

### タイプ② 改善志向型エンジニアタイプ（Improver）
- **キャッチ:** 仕組みを変えたい、課題解決型
- **説明:** 業務の非効率や無駄に気づきやすく、仕組みを良くしたい気持ちが強いタイプ。システム開発や業務改善に適性がある。
- **向いている職種:** Webエンジニア・業務システム開発・DX推進エンジニア

### タイプ③ 自由志向型エンジニアタイプ（Flexible）
- **キャッチ:** 場所も時間も、自分らしく働きたい型
- **説明:** 場所や時間に縛られない働き方を重視するタイプ。リモートワーク中心のIT業界と相性が良い。
- **向いている職種:** フリーランスエンジニア・フルリモート企業のエンジニア

### タイプ④ 成果実感型エンジニアタイプ（Achiever）
- **キャッチ:** 頑張りが報われる環境で、本気を出せる型
- **説明:** 実力や努力が正当に評価される環境にモチベーションを感じるタイプ。成果が見えやすいIT業界と相性が良い。
- **向いている職種:** スタートアップエンジニア・フリーランス・事業会社エンジニア

## LINE登録ゲート仕様

### STEP3（簡易フィードバック）の表示内容

```
🎯 診断が完了しました

エンジニア適性スコア: [スコア]点

あなたには高いエンジニア適性が確認されました。
詳しい診断結果（タイプ・向いている職種・次のステップ）は
LINE公式アカウントで受け取れます。

[LINEで受け取る →]（#06c755 緑ボタン）

※ 登録後すぐに結果が届きます
```

- LINEボタンのリンク先: LINE友だち追加URL（`https://lin.ee/XXXXXXX`）
- URLにタイプパラメータを付与して診断タイプを保持する方法は採用しない（LINE連携が複雑になるため）
- LINE自動返信メッセージに `/result?type=I` 形式のURLを4タイプ分用意してもらう運用とする

### STEP5（診断結果）の表示内容

- URLパラメータ `type`（S / I / F / A）を受け取って対応するタイプを表示
- `type` 未指定またはが無効値の場合は「改善志向型（I）」をデフォルト表示
- CTA: 「無料カウンセリングに申し込む →」ボタン（外部リンク）

## LINE側の運用設定（アプリ外の作業）

友だち追加時の自動返信メッセージに以下の4URLを案内する（LINEの管理画面で設定）。

```
タイプS: https://[ドメイン]/result?type=S
タイプI: https://[ドメイン]/result?type=I
タイプF: https://[ドメイン]/result?type=F
タイプA: https://[ドメイン]/result?type=A
```

> ※ 現時点ではユーザーが自分のタイプを把握した上でLINE登録するため、「あなたのタイプの結果はこちら」のメッセージに4つのURLを全部掲載する形が最もシンプル。

## 非機能要件

- **モバイルファースト:** スマホ（375px〜）で快適に操作できること
- **表示速度:** Vercel + 静的生成により初期表示 1秒以内
- **アクセシビリティ:** 選択肢はボタン要素を使用し、キーボード操作可能
- **SEO:** TOP ページに基本的な meta タグを設定する
